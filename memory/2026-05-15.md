## Claw3D Loading Fix on JormUI (04:15-04:30)

**Problem:** Claw3D (3D Office tab) wasn't loading in the Perfect Dashboard (JormUI).

**Root Cause:** Multiple issues with the Claw3D proxy in `server.js`:

1. **Neither server was running** — Both the dashboard (port 3000) and Claw3D (port 3002) Node processes were down. Started both.

2. **Redirect loop** — Claw3D's Next.js server returns a 307 redirect from `/` → `/office`. The proxy was passing this redirect through unchanged, so the browser would follow it to `/office` (the dashboard's own Agent Office page) instead of staying within the proxy path.

3. **Missing asset proxy** — Claw3D's HTML references assets at absolute paths like `/_next/static/chunks/...`. When loaded through the proxy, the browser requested these from the dashboard server which doesn't have them.

4. **Missing WebSocket proxy** — Claw3D connects to OpenClaw Gateway via WebSocket at `/api/gateway/ws`. This upgrade request needed to be proxied too.

**Fixes Applied to `server.js`:**

1. **Redirect rewriting** — Added Location header rewrite in the `/claw3d-proxy` middleware: `Location: /office` → `Location: /claw3d-proxy/office`

2. **Asset proxy middleware** — Added a new middleware that catches `/_next/static/*`, `/favicon.ico`, and `/assets/*` paths and proxies them to Claw3D on port 3002.

3. **WebSocket upgrade proxy** — Added `server.on('upgrade', ...)` handler that intercepts WebSocket upgrade requests to `/api/gateway/ws` and proxies them to Claw3D.

**Files Modified:**
- `C:\Users\kency\perfect-dashboard\server.js` — All three fixes

**Servers Status:**
- Dashboard: port 3000 ✅
- Claw3D: port 3002 ✅
- Proxy: working ✅

## Claw3D Production Build Fix (19:41-20:00)

**Problem:** Claw3D was still not working after the earlier proxy fixes. The 3D Office tab showed "Opening Claw3D..." but the connection failed with "Gateway closed (1006):".

**Root Cause:** The Claw3D Next.js dev server (PID 8636) was stuck — it had a lock file issue (`.next\dev\lock`) from a previous crash and was unable to acquire the dev lock. The server process was running but not serving any requests. All HTTP requests to port 3002 timed out.

**Fixes Applied:**

1. **Killed stuck dev server** — `taskkill /PID 8636 /F`
2. **Removed lock file** — Deleted `C:\Users\kency\claw3d\.next\dev\lock`
3. **Built production version** — `npx next build` completed successfully (55s compile + 32 pages generated)
4. **Started production server** — `node server/index.js` (no `--dev` flag)

**Key Findings:**
- The dev server was compiling `/office` page and got stuck on Tailwind CSS processing
- Production build resolved the compilation issue
- The gateway (port 18789) was always working fine — the issue was solely on the Claw3D side
- WebSocket connection from Claw3D browser to `ws://localhost:18789` works correctly
- The `resolveStudioProxyGatewayUrl` function correctly returns `ws://localhost:18789` for loopback hosts
- The `NEXT_PUBLIC_GATEWAY_URL=/api/gateway/ws` is a relative path that resolves to Claw3D's own proxy, which then forwards to the upstream gateway
- Gateway WS endpoint is at root `/` (accepts WebSocket upgrades) and also at `/gateway/ws` and `/ws`
- `/api/gateway/ws` on port 18789 returns 404 — this is NOT the gateway's WS endpoint
