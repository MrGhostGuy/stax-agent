## v4.0 — AI Agent Gateway (Built & Deployed This Session)

### What Was Built
- **ReAct-style autonomous agent** with tool-calling capabilities
- Agent tools: `web_search` (DuckDuckGo), `web_fetch` (read any URL), `calculate` (safe math)
- Agent memory: conversation history persisted per API key (last 6 turns, stored in JSONBlob)
- Up to 5 LLM round-trips per agent request using OpenAI `tool_calls` format
- Two access patterns:
  - `POST /v1/agent` — explicit agent endpoint
  - `POST /v1/chat/completions` with `agent: true` — agent mode via chat endpoint

### Tier Changes for v4.0
- Free ($0/14 req): Chat only, NO agent
- Pro ($6/40 req): ✅ Full agent with all tools
- Premium ($27/500 req): ✅ Agent+ with all tools

### Bugs Fixed During Development
1. **Hardcoded secret in code** — GitHub push protection blocked commit. Removed hardcoded key, use only env var.
2. **Vercel env var set to "y"** — Interactive `vercel env add` only captured the "is sensitive?" confirmation. Fixed by removing and re-adding properly.
3. **JSONBlob missing `agent_history` field** — Old blob didn't have the new field. Fixed `loadDb()` to merge blob data with default schema.
4. **Agent tier check missing on `/v1/agent`** — Free users could access agent. Added `allows_agent` check in `handleAgent()`.
5. **`finalHistory` typo** — Referenced undefined variable in max-iterations fallback. Fixed to `finalContent`.
6. **Tool message format** — Ensured assistant message with tool_calls is properly preserved when pushing to conversation history.

## v5.0 — Complete Platform Upgrade (Built & Deployed This Session)

### New Features
- **Stripe Checkout integration**: Pro/Premium signups create Stripe Checkout sessions; users start as free until payment confirmed
- **Subscriber Dashboard** (landing page tab): Enter API key → view usage, manage keys, reset agent memory
- **Key Management API**: Create additional keys, rotate (regenerate), revoke. Multiple keys per account.
- **Proper rate limiting**: Per-minute and per-day rate limits enforced per tier, persisted in JSONBlob
- **Account consolidation**: Keys linked by email, can manage all keys from one account
- **Success/cancel URL handling**: Stripe redirect back to landing page with ?success=true or ?canceled=true

### Key API Additions
- `POST /v1/keys` — Create additional API key (body: {label, tier})
- `GET /v1/keys` — List all keys for account
- `DELETE /v1/keys/:prefix` — Revoke a key
- `POST /v1/keys/:prefix/rotate` — Rotate (regenerate) a key
